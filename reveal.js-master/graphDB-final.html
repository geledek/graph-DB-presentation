Limitations of Traditional SQL Databases

  • Data dispersed across multiple tables.
  • Need to perform multiple joins to retrieve whole data. Computationally expensive for large datasets.
  • Enforcing constraints for update/insert/delete.
  • Above all - burden for programmer due to complex design.

NoSQL - The way forward

  • NoSQL= Not Only SQL. [SQL + a whole lot more]
  • 4 categories of NoSQL databases - 
    • Key-value stores.  Eg - Redis
    • Column Families. Eg - Cassandra
    • Document Databases. Eg - MongoDB
    • Graph Databases. Eg - Neo4J

Introducing Graphs

  • Graph - Abstract representation of a set of 'objects'.
  • A pair of objects may be connected by 'links'.

Graph Databases

  • A database with an explicit graph structure.
  • Each node knows all its adjacent nodes -> no index required.
  • Advantages:
    • closer to real world object representation.
    • no joins -> better scalability.
    • constant time retrieval, irrespective of depth or breadth.
    • No rigid schema -> better flexibility in design. 

Typical functionalities supported

  • Data storage (in memory/external memory)
  • Multiple formats for graph representation.
  • DDL, DML, DQL
  • Adjacency, Reachability, Pattern Matching, Summarization & Aggregation query support.
  • External APIs.

Applications

  • Social networking.
  • Recommendations/ Match-making apps.
  • Path finding (Eg - Linkedin connections).
  •  Shortest path (travelling salesman problem).
